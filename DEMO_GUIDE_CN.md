# Xhell 项目演示流程指南 (新版界面)

本文档适用于新的 **"指令面板"** 风格界面。现在的演示流程非常简单，全程只需鼠标点击。

## 演示前准备

1. **一键启动**：
   在终端运行：
   ```bash
   ./start_demo.sh
   ```
2. **进入网页**：浏览器访问终端显示的 URL (例如 `http://localhost:8501`)。

---

## 演示流程

### 1. 基础命令演示

**位置**：左侧侧边栏 -> **📂 1. 基础命令** 区域

1. **查看路径**: 点击 **[1-1. 查看当前路径 (xpwd)]**
   - *观察*: 中间终端窗口显示 `/.../demo_workspace`
2. **创建文件**: 点击 **[1-4. 创建测试文件]**
   - *观察*: 
     1. 终端显示 `xtouch test.txt` 执行成功。
     2. **右侧 "工作区文件监控" 面板会自动刷新，出现 `test.txt`。**
     3. 选中右侧的 `test.txt`，可以看到它是空的。
3. **查看目录**: 点击 **[1-2. 列出文件 (xls)]**
   - *观察*: 终端列出 `test.txt`。

#### 进阶文件操作 (新)
4. **复制文件**: 点击 **[1-5. 复制 (xcp)]** -> 创建 `src.txt` 并复制为 `dst.txt`。
5. **移动/重命名**: 点击 **[1-6. 移动 (xmv)]** -> 创建 `old.txt` 并重命名为 `new.txt`。
6. **删除文件**: 点击 **[1-7. 删除 (xrm)]** -> 创建 `junk.txt` 并删除它。

### 2. I/O 重定向演示

**位置**：左侧侧边栏 -> **📝 2. I/O 重定向** 区域

1. **写入文件**: 点击 **[2-1. 写入文件 (>)]**
   - *观察*: 
     1. 执行 `xecho ... > hello.txt`。
     2. 右侧文件列表出现 `hello.txt`。
     3. **点击右侧 `hello.txt`**，下方立即显示内容 "Hello Xshell"。
2. **追加内容**: 点击 **[2-2. 追加内容 (>>)]**
   - *观察*: 
     1. 执行追加命令。
     2. 右侧文件内容自动更新，增加了 "New Line"。

### 3. 管道功能演示

**位置**：左侧侧边栏 -> **🔗 3. 管道操作** 区域

1. **统计计算**: 点击 **[3-1. 统计单词 (wc -w)]**
   - *观察*: 终端显示命令 `xecho ... | wc -w`，输出结果 `4` (已修复计数 Bug)。
2. **组合命令**: 点击 **[3-2. 统计文件数 (wc -l)]**
   - *观察*: 统计当前目录下有多少个文件。
3. **双向输出**: 点击 **[3-3. 双向输出 (xtee)]**
   - *观察*: 输出同时显示在屏幕上，并写入 `tee_out.txt`。

### 4. 历史记录与清理

**位置**：左侧侧边栏 -> **⚙️ 4. 系统管理** 区域

1. **查看历史**: 点击 **[4-1. 查看历史 (xhistory)]**。
2. **查看日志**: 点击 **[4-2. 系统日志 (xjournalctl)]** -> 查看 Xhell 内部运行日志。
3. **一键重置**: 
   - 点击 **[4-3. 清空环境 (Reset)]**。
   - *效果*: 所有演示文件被删除，历史记录清空，界面恢复初始状态。

### 5. 扩展功能演示 (加分项)

**位置**：左侧侧边栏 -> **🌟 5. 扩展功能** 区域

1. **系统信息**: 点击 **[5-1. 系统信息 (xsysinfo)]**
   - *观察*: 终端显示真实的 Linux 内核版本、CPU 型号和内存使用情况。这是通过读取 `/proc` 文件系统实现的。
2. **查看帮助**: 点击 **[5-2. 帮助菜单 (xhelp)]**
   - *观察*: 列出 Xhell 支持的所有 15 个命令及其说明。

## 常见问题

- **界面无反应？**: 尝试点击右下角的 **[🔄 刷新文件列表]** 或刷新浏览器。
- **报错 Not Found？**: 请确保是通过 `./start_demo.sh` 启动的，脚本会自动处理路径问题。
